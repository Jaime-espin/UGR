# Análisis de Rendimiento de Algoritmos - Práctica ALG

## Descripción del Proyecto
Proyecto educativo de análisis de rendimiento para 9 algoritmos en C++. Los algoritmos miden su tiempo de ejecución usando la librería `<chrono>` de C++ y generan datos CSV para análisis comparativo con compañeros de clase.

## Estructura de Carpetas
```
src/              - 9 archivos C++ con algoritmos
  ├── burbuja.cpp
  ├── insercion.cpp
  ├── seleccion.cpp
  ├── mergesort.cpp
  ├── quicksort.cpp
  ├── floyd.cpp
  ├── dijkstra-iterado.cpp
  ├── fibonacci.cpp
  └── hanoi.cpp
scripts/          - Scripts bash para automatización
docs/             - Documentación técnica
datos/            - Archivos CSV generados (salida)
```

## Algoritmos
- **Cuadráticos (O(n²))**: burbuja, inserción, selección
- **O(n log n)**: mergesort, quicksort
- **Cúbicos (O(n³))**: floyd, dijkstra-iterado
- **Exponenciales**: fibonacci (recursivo), hanoi

## Especificaciones Técnicas
- **Lenguaje**: C++11+
- **Timing**: `std::chrono::high_resolution_clock` (microsegundos)
- **RNG**: `srandom()` + `random()` (POSIX para Linux)
- **Salida**: CSV formato `n,tiempo_en_segundos`
- **Compilación**: `g++ -O2`
- **Plataforma**: Linux (no Windows)

## Reglas Críticas
⚠️ **IMPORTANTE**: Los algoritmos NO deben ser modificados. Solo se añadió `#include <chrono>` y un wrapper de medición de tiempo alrededor del algoritmo principal. Las variables y lógica del algoritmo deben permanecer idénticas al original.

## Scripts Disponibles
- `generar_datos.sh` - Genera todos los CSV con rangos de n optimizados por complejidad
- `prueba_rapida.sh` - Ejecución rápida para verificación
- `ejecutar_pruebas.sh` - Script original en raíz

## Compilación y Ejecución
```bash
# Compilar algoritmo individual
g++ -O2 src/burbuja.cpp -o burbuja
./burbuja 100000

# Ejecutar script de automatización en Linux
bash ./generar_datos.sh
```

## Consideraciones Especiales
- Los archivos usan `srandom()/random()` que es POSIX-específico de Linux
- El proyecto está optimizado para ser reproducible en Linux
- Los datos CSV se usan para comparar resultados con compañeros
- Asegúrate de que los algoritmos permanecen intactos al hacer cambios
